VCS (Git, Github, gh-pages)

git config --global user.name blabla - налаштування даних користувача - імені git config --global user.email blabal@bla - налаштування даних користувача - емейл. git init - створення (ініціалізація) нового репозиторію git status - показує інформацію про поточний стан файлів у робочій директорії git add - додавання файлу (-ів) робочої директорії до індексації (staging area) git commit - створення комміту (зліпку проекту) Зберігає дані, які були внесені до індексу командою $git add. Аргумент "-m" дозволяє одразу ввести коментар про зміну git log - перевірка історії комітів git checkout - команда для створення гілок, переміщення між гілками (та копіювання файлів із вибраної гілки до робочої директорії) git branch - команда для роботи із гілками репозиторію (створення, видалення, вивід списку гілок).без жодних параметрів - виводить список наявних гілок. git push - перенесення даних з локального репозиторію на віддалений git pull - стягнення данних з віддаленого репозиторію на локальний та їх мердж. (є обгорткою для команд $git fetch та $git merge) git merge - злиття гілки(-ок) із активною поточною гілкою git clone - клонування віддаленого репозиторію у локальну директорію. команда є обгорткою для кількох команд - 1) створює новий репозиторій у локальній папці ($git init), прив'язує його до віддаленого репозиторію ($git add origin [URL]), скачує дані з віддаленого репозиторію до локального ($git fetch) та обновлює до останнього коміта ($git checkout). .gitignore файл - файл, в який можна внести список файлів та папок, які не будуть індексуватись гітом (будуть приховані від git). Advanced git revert - команда для можливості відкату до одного з попередніх комітів. Створює новий коміт та поміщає в ньому потрібний стан та зміни файлів. git reset - команда для відкату до обраного коміта. !Видаляє зміни, що були зроблені після обраного коміту. git rebase - команда для злиття однієї гілки з іншою, але напротивагу команді $git merge переписує історію комітів. git cherry-pick - команда, що дозволяє взяти зміни з якогось коміту (напр. з іншої гілки, без повного злиття з нею) та застосувати ці зміни у вигляди нового коміту у поточній гілці. git stash - команда, що дає можливість тимчасово зберегти у пам'яті незакомічені зміни у поточній гілці, зі змогою повернутись до роботи з ними через певний час.
